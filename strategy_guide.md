## 👥 Role Definitions: Who Does What?

### 1. What You (Yatharth) Do:
* **The Director:** You provide high-level goals to Hermes in the `#sprint-main` Slack channel.
* **The Gatekeeper:** You review OpenClaw's PRs in `#human-review`. **You must manually merge all PRs on GitHub.** No bot is permitted to merge to `main`.
* **The Auditor:** You capture evidence screenshots throughout the day and run the final Slack export.

### 2. What the AI Agents (Hermes & OpenClaw) Do:
* **Hermes (Product Owner):** Decomposes your goals into backlog issues, saves sprint markdown files (e.g. `sprints/sprint-01.md`), and tags OpenClaw in `#agent-coder` to assign tasks.
* **OpenClaw (Coder):** Creates files, writes PHP/JS code, runs local tests, commits changes, opens PRs, and reports status in `#agent-log` using the *What I Did / Left / Call* structure.

### 3. What I (Antigravity - Your Co-Pilot) Do:
* **The Architect:** I write playbooks, technical blueprints, and database schemas.
* **The Debugger:** If OpenClaw hits a coding bug, test failure, or environment error, paste the logs to me here in this IDE chat. I will write the fix/refactoring script for OpenClaw to execute.
* **The Prompt Engineer:** I will help you write the exact prompts to redirect the agents if they get stuck or veer off-track.

## 📋 Pre-Start Setup Checklist
Before starting Sprint 1, execute these steps exactly:
1. **GitHub Setup:** Create a fresh, empty public repository on GitHub named `forge2/<yourname>`.
2. **Local Repo:** Clone the empty repository to `C:\Desktop\forge 02`. Download and unzip the hackathon skeleton starter kit inside this directory.
3. **Slack Setup:** Ensure these 5 public channels are created in your workspace:
   * `#sprint-main` (Human ⇄ Hermes planning)
   * `#agent-coder` (Hermes ⇄ OpenClaw task handoffs)
   * `#agent-log` (OpenClaw status reports)
   * `#ci-cd` (CI results)
   * `#human-review` (PR reviews and approvals)
4. **Boot Gateways:** In separate PowerShell windows, start the gateways:
   * Run `openclaw gateway` in Terminal 1.
   * Run `hermes gateway` in Terminal 2.

## 💾 Deliverables & Evidence Log Checklist
The judges clone your repository and build three timelines: Slack export, Git commits, and CI runs. They must match. Here is the checklist of deliverables that must be committed inside your repository:

### 1. Agent Configurations (`/agents`)
* `agents/hermes/hermes-config.yaml` — Your Hermes configuration file (API token redacted to `${SLACK_BOT_TOKEN}`).
* `agents/openclaw/openclaw.json` — Your OpenClaw configuration file (API token redacted to `${EASTROUTER_API_KEY}`).

### 2. The Sprint Backlogs (`/sprints`)
* `sprints/sprint-01.md` through `sprints/sprint-04.md` — Backlog files containing issues, tasks, and outcomes for each sprint (generated by Hermes).

### 3. The Agent Loop Log (`/agent-log.md`)
* `agent-log.md` (at repo root) — A chronological log of every human ➔ Hermes ➔ OpenClaw handoff, along with status reports.

### 4. Slack Workspace Export (`/slack-export`)
* **Real Slack Export (Preferred):** Export data from Slack Workspace Settings (date range: "Entire history"). Unzip the `.zip` file and commit the raw JSON files and channel folders directly inside `slack-export/`.
* **Backup Screenshots:** If export fails, commit screenshots of each channel showing real messages and timestamps to `slack-export/screenshots/`.

### 5. Screenshots (`/evidence/screenshots`)
Name the PNGs clearly (e.g. `01-dashboard-view.png`) showing:
* **The App:** Ticket list queue, ticket conversation thread showing customer replies vs internal notes, and the metrics dashboard.
* **Multi-tenancy Proof:** Two screenshots of the same page logged in as Organization A vs Organization B showing completely different data.
* **Terminals:** Screenshots showing both `openclaw gateway` and `hermes gateway` running in your terminal.
* **CI:** Screenshot showing a green run of your GitHub Actions CI check.
